# expected/ — materialized outputs (placeholders until Phase 2)

The files in this directory are the **materialized outputs** for the `association-board-brief`
example: `nodes.json`, `edges.json`, and `state.json` (the graph and release state derived from
`../events.jsonl`).

- They are **generated by `pgf materialize`** (the Phase 2 CLI) and committing the regenerated golden
  copies is a **Phase 4 deliverable**.
- They are currently **intentionally empty placeholders** — `nodes.json` is `[]`, `edges.json` is
  `[]`, and `state.json` is `{}` — carried over from the Phase 0 skeleton, because the materializer
  does not exist yet. They are **not** hand-authored: authoring graph output by hand would invent the
  tool's result.
- **No test may compare against these fixtures until they are regenerated by `pgf materialize`.**
  Comparing now would bless an empty graph against a 46-event log (the current `events.jsonl`
  materializes to roughly 23 nodes and 10 edges), which would be a false pass.
- **Phase 2's first materializer milestone regenerates these files** before any comparison/golden test
  is wired into CI.

The `release-package/` subdirectory — and any release-package digests (`file_digests`) — are likewise
Phase 4 `pgf export` outputs: empty/placeholder until the exporter exists, consistent with the
`nodes.json` / `edges.json` / `state.json` placeholders above. No release package is materialized in
Phase 1.
